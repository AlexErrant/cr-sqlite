Official SQLite WASM Build

  • Main Thread (in memory)
  • On Worker (with persistence)

WA-SQLite

  • Main Thread (with persistence)
  • Worker Thread (with persistence)
  • Shared Worker Exploration (does not work -- used for development purposes)

What's the difference between the two?

Official Build:

Pros:
  • Permissive license (public domain)
  • Fastest
  • Supported by the SQLite team
Cons:
  • Poor support for concurrent access to same DB file
  • Requires COOP & COEP headers
Neutral:
  • Must run in a worker for persistence but worker comms can be abstracted over.

wa-sqlite Build:

Pros:
  • Can persist on main thread
  • Does not require COOP & COEP headers
  • Supports high concurrent access to same DB file
Cons:
  • GPLv3 License
  • Slower
